### 1.3 LoRA 微调的参数量记忆定律

浙江大学团队发表论文 **《How LoRA Remembers? A Parametric Memory Law for LLM Finetuning》**，首次系统量化 LoRA 的参数记忆容量：

- 发现 **Parametric Memory Law**：损失下降 ΔL 与有效参数量和序列长度之间存在稳健的幂律关系
- 在 token 级别揭示**确定性相变**：预测概率 p > 0.5 是贪婪解码下逐字回忆的充分条件
- 提出 **MemFT**：阈值引导的优化策略，动态将训练预算重新分配到 sub-threshold token

**💡 对你的价值**：如果你在用 LoRA 做模型微调，这篇论文给出了精确的记忆容量预测方法，可以帮你优化训练策略，避免浪费算力。

## 二、Agent 架构与范式

### 2.1 多组件 Agent 的组合不一致性问题

论文 **《Locally Coherent, Globally Incoherent: Bounding Compositional Incoherence in Multi-Component LLM Agents》**（ICML 2026 Workshop）揭示了一个关键问题：

**问题定义**：多组件 LLM Agent 中，每个组件在局部视角下可能做出合理推断，但组合后可能违反基本概率公理。

**关键发现**：
- 在 1,876 个集成子句上，**33-94%** 的子句存在组合不一致（ε* > 0）
- 导致每笔投注 +0.115 nats 的遗憾（在比例分配规则下）
- 三种直觉性缓解策略（检索、分区感知提示、聚合器 LLM）**均失败或回退**

**解决方案**：
- 定义**组合残差 ε***：运行时可计算的 L2 距离
- **分层 Boyle-Dykstra 投影**：确定性修复组合不一致
- **e-process**：随时有效的序列一致性监控

**💡 对你的价值**：如果你正在构建多 Agent 系统或工具调用链，这篇论文提供了在运行时检测和修复不一致的数学工具，直接关系到系统的可靠性。

### 2.2 潜伏推理：用工作记忆代替自回归推理

论文 **《Unlocking the Working Memory of Large Language Models for Latent Reasoning》** 提出了 RiM（Reasoning in Memory）方法：

**核心思想**：人类认知可以在工作记忆中操作信息而不需要外化中间思考。RiM 用**记忆块**（固定特殊 token 序列）替代自回归生成的推理步骤。

**技术细节**：
- 记忆块是固定的而非生成的，可以在**单次前向传播**中处理
- 两阶段课程学习：(1) 通过预测显式推理步骤来锚定记忆块 (2) 丢弃步骤级监督，在每个记忆块后迭代精炼答案
- 跨不同模型族和大小的语言模型，RiM **匹配或超过**现有潜伏推理方法

**与现有方法对比**：

| 方法 | 推理方式 | 计算效率 | 外化思考 |
|------|----------|----------|----------|
| 标准 CoT | 自回归生成 | 低（多步生成） | 是 |
| 潜伏推理（现有） | 隐含 token | 中 | 否（但仍有自回归开销） |
| RiM（本文） | 记忆块 | 高（单次前向传播） | 否 |

**💡 对你的价值**：如果你关心推理成本，RiM 提供了一种显著降低推理时计算量的思路——特别是对于需要深度推理但不需要暴露推理链的场景。

### 2.3 预规划推理框架 PPC

论文 **《Knowing What to Solve Before How: Preplan Empowered LLM Mathematical Reasoning》** 提出 PPC（Preplan-Plan-CoT）框架：

**核心洞察**：现有 plan-based 方法都在解决"怎么做"，但忽略了一个前置问题——**"解什么"**（识别问题类型、可用工具、可预见陷阱）。

**架构**：
```
问题 → 预规划(Preplan) → 规划(Plan) → 思维链(CoT)
        ↑ 识别问题类型     ↑ 制定解法     ↑ 执行求解
```

**关键设计**：
- 三阶段合成流水线，包含 spoiler-score 检测器过滤泄露和 spoiler 失败
- 复合 GRPO 奖励：确保生成的规划真正遵循预规划
- 在 4 种骨干模型、5 个数学推理基准上，**40 项指标中 39 项最佳**
- maj@16 提升 +2.23，pass@16 提升 +3.06，**不增加推理 token 开销**

**💡 对你的价值**：PPC 的思路可以应用到非数学场景——先让模型"理解问题本质"再制定解决方案，这对复杂工作流的 Agent 设计有启发。

## 四、AI 工具与技巧

### 4.1 测试时微调新方案：HullFT

论文 **《Efficient Test-Time Finetuning of LLMs via Convex Reconstruction and Gradient Caching》** 提出 HullFT：

**问题**：测试时微调（TTFT）需要在每个查询时快速选择相关样本并微调，但现有方法在速度和质量之间存在 trade-off。

**HullFT 方案**：
1. 使用 Frank-Wolfe 优化将查询嵌入表示为训练序列的**稀疏凸组合**
2. 得到的支持集天然相关且多样
3. 通过几何整数化将分数权重转换为精确整数多重集
4. **梯度重用**：在重复的微调步骤间摊销前向-后向计算

**效果**：相比 SOTA TTFT 方法，在更低总运行时间下实现更低的 bits-per-byte。

**💡 对你的价值**：如果你在考虑为每个查询动态微调模型（比如定制化客服、个性化助手），HullFT 提供了更高效的实现方式。

### 4.2 上下文奖励自适应：让 RLHF 适应多样化偏好

论文 **《In-Context Reward Adaptation for Robust Preference Modeling》** 解决了 RLHF 中奖励模型的刚性问题：

**问题**：传统 RLHF 使用静态奖励模型，但人类价值观是多样且异质的。

**方案**：利用 Transformer 的上下文学习能力，从少量偏好示范中自适应推断底层奖励结构。关键发现是：加入**人类响应时间**作为辅助输入信号后，模型能够成功适应来自未见领域的偏好。

**💡 对你的价值**：如果你在做模型对齐或偏好优化，这个思路可以帮你构建更灵活的奖励系统。

### 4.3 多轮对话中的"自我锚定漂移"问题

论文 **《Same Evidence, Different Answers: Canonical-Context On-Policy Distillation for Multi-Turn Language Models》** 揭示了一个常见但被忽视的问题：

**问题**：LLM 在单轮提示中给出所有指令时能完成任务，但在相同信息跨多轮逐步透露时会失败。原因是**自我锚定漂移**——部分信息下产生的回应引入了无根据的假设，这些假设后来扭曲了最终答案。

**解决方案 CCOPD**：
- 冻结教师模型在完整提示上操作
- 训练学生模型在多轮对话中接收增量证据
- 将学生的行为与教师的规范完整上下文行为对齐

**效果**：在多轮数学对话上训练后，跨数学和 5 个零样本域外任务族，RAW-SHARDED 性能平均相对提升 **32%**。

**💡 对你的价值**：如果你在做多轮对话系统或聊天机器人，这个发现解释了为什么对话式交互比单轮提示更容易出错，并给出了训练级解决方案。

### 4.4 长文档翻译 Agent：Loong

论文 **《A Human-Like Long Document Translation Agent with Observe-and-Act Adaptive Context Selection》** 提出 Loong：

**核心设计**：
- **3E 记忆模块**：Essence（摘要）- Exemplar（句对）- Entity（实体记录）
- 不被动关注所有历史，而是通过深度推理自适应识别最佳上下文
- 通过强化学习优化上下文策略，使用自身采样的观察-行动推理轨迹生成偏好数据

**效果**：英↔中、德、法翻译平均提升最高 **13.0 分**，在超长文档翻译中保持稳定。

**💡 对你的价值**：如果你有长文档翻译需求，Loong 的 3E 记忆设计思路可以借鉴到任何长上下文 Agent 场景中。

### 4.5 模型数据混合审计：LLMSurgeon

论文 **《Diagnosing Data Mixture of Large Language Models》**（ACL 2026 Main）提出：

**问题**：LLM 的预训练数据混合构成其"数字 DNA"，但这种组合很少公开，使得事后审计困难。

**LLMSurgeon**：
- 仅通过目标 LLM 生成的文本，估计其预训练语料库的领域级分布
- 将数据混合手术（DMS）公式化为标签偏移假设下的逆问题
- 估计校准的**软混淆矩阵**并求解约束逆问题

**💡 对你的价值**：如果你是 AI 研究者或监管人员，这个工具可以帮你审计闭源模型的训练数据构成。

## 五、值得深读的研究

### 5.1 自我训练的验证器：解锁训练和测试时自我改进

📄 **《Self-Trained Verification for Training- and Test-Time Self-Improvement》**

**研究方法**：
- 核心观察：模型单独无法捕捉自身生成的错误，但在看到参考答案时可以
- 将这种不对称性转化为监督目标，训练验证器模仿更知情的自身版本
- 提出 **ViL**（Verifier-in-the-loop Training）：在 V-R 循环内使用 STV 验证器的反馈进行 RL 训练

**核心发现**：
- 测试时：STV 在高难度数学问题上将准确率**翻倍**，在科学推理任务上提升 **14 倍**（1.5% → 21%）
- 训练时：从 RL 收敛的生成器出发，ViL 在 pass@1 上获得**额外 33% 增益**
- 更关键的是：生成器的独立 pass@1（测试时无验证器）超过标准 RL 收敛点 **30% 相对增益**

**💡 对你的价值**：这篇论文指向了推理模型训练的下一个前沿——如何训练验证器以及如何与验证器一起训练。如果你在微调推理模型，这个思路可能比单纯增加训练数据更有效。

### 5.2 稀疏奖励下的 GRPO 改进：HPO

📄 **《Hysteretic Policy Optimization for Stable and Efficient Training under Sparse-Reward Regime》**

**研究方法**：
- 发现 GRPO 在稀疏可验证奖励下的失败模式：早期更新中负优势响应多于正优势响应
- 提出 **HPO**：减少负优势更新的权重，将逐响应长度归一化替换为平均长度归一化
- 进一步提出 **A-HPO**：基于批次级优势符号统计自动设置滞后权重

**核心发现**：
- 在 TeleLogs 上，A-HPO 最终奖励 0.84，超越 SAPO 5%、GSPO 11%、GRPO 15%
- 在 Countdown 上，在 1.5B-7B 模型的最难配置中获得最大增益
- 消融研究表明增益来自更好地平衡正负优势的贡献

**💡 对你的价值**：如果你在用 GRPO 做强化学习微调（特别是稀疏奖励场景），A-HPO 是一个简单但有效的改进方案。

### 5.3 中期训练的数据选择：MIRA

📄 **《Mid-training Rubric Anchoring for Source-Aware Data Selection》**

**研究方法**：
- 中期训练的数据选择问题独特：数据在近预训练规模下优化，但面向下游能力且来自异构来源
- 提出 **MIRA**：基于自锚定评分发现的数据过滤框架
- 关键思路：让评分标准构建成为数据选择的一部分
- 在代码导向的中期训练（21 个来源，5 个来源组）上，MIRA 在 9 个代码基准上超过选择基线

**核心发现**：使用仅**一半的 token** 即可匹配完整语料库的运行效果。

**💡 对你的价值**：如果你在训练或微调 LLM，MIRA 的思路可以帮你用更少的数据获得更好的效果，节省算力成本。

### 5.4 多模态基准测试的统计分辨率问题

📄 **《Resolution Diagnostics for Paired LLM Evaluation》**（ICML 2026 Workshop）

**研究方法**：
- 将配对 LLM 评估公式化为假设检验问题
