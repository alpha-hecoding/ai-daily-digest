## 二、Agent 架构与范式

### 2.1 生产级 LLM Agent 的运行时架构模式选择方法论 ⭐ 必读

**论文：** arXiv:2605.20173  
**作者：** Vasundra Srinivasan  
**日期：** 2026 年 5 月 19 日

#### 研究方法

这篇论文提出了一个对构建生产级 Agent 极具指导意义的框架。作者首次命名了 **随机-确定性边界（Stochastic-Deterministic Boundary, SDB）**——这是 LLM Agent 架构中最核心的概念。

SDB 被定义为一个四部分契约：
1. **Proposer（提议者）：** LLM 生成候选行动
2. **Verifier（验证者）：** 确定性系统验证提议的可行性
3. **Commit Step（提交步骤）：** 将验证通过的行动执行到系统中
4. **Reject Signal（拒绝信号）：** 当行动失败时反馈给 LLM

围绕这个核心原语，作者将 Agent 运行时设计组织为三个关注点：
- **Coordination（协调）：** Agent 之间如何协作
- **State（状态）：** 如何管理 Agent 的长期记忆和上下文
- **Control（控制）：** 如何调节 Agent 的行为和决策流程

#### 六种运行时模式

论文提供了六种可组合的运行时模式：

| 模式 | 适用场景 | SDB 强度 | 源自 |
|------|---------|---------|------|
| 分层委派（Hierarchical Delegation） | 复杂多步任务分解 | 强 | 分布式系统 |
| 散播-聚集+Saga（Scatter-Gather + Saga） | 并行探索后聚合 | 中 | 微服务事务 |
| 事件驱动排序（Event-Driven Sequencing） | 实时响应系统 | 强 | 事件溯源 |
| 共享状态机（Shared State Machine） | 多 Agent 协作 | 强 | 状态机理论 |
| 监督者+门控（Supervisor + Gate） | 质量保证关键场景 | 最强 | 质量控制 |
| 人在回路（Human in the Loop） | 高风险决策场景 | 可变 | HCI |

#### 核心发现

1. **重放发散（Replay Deterministic Event Log）：** 当 LLM 消费确定性事件日志时，模型版本或提示词的变化会导致不同的下游输出。这是一个全新的故障模式，传统软件工程中不存在。

2. **可靠性分解：** 每调用的模型方差 vs 架构动量。随着模型方差降低，模式选择和 SDB 强度成为长期可靠性的越来越重要的杠杆。

3. 论文提供了**五步方法**来选择运行时模式，以及将生产故障映射到模式弱点的诊断程序。

#### 启发

如果你的 Agent 在生产环境中出现"同样的输入，有时成功有时失败"的问题，很可能就是**重放发散**。解决方案不是更换模型，而是**增强 SDB**——在 LLM 输出和系统行动之间增加更强的验证层。

#### 💡 对你的价值

这是目前关于生产级 Agent 架构最系统的理论框架。如果你正在构建任何需要可靠运行的 Agent 系统（不只是 demo），强烈建议阅读完整论文（25 页，6 个表格，附参考实现）。

**参考代码：** [github.com/vasundras/agent-runtime-patterns](https://github.com/vasundras/agent-runtime-patterns)

### 2.2 POW3R：策略感知的评分量奖励框架

**论文：** arXiv:2605.20164  
**作者：** Utkarsh Tyagi 等  
**日期：** 2026 年 5 月 19 日

#### 研究方法

在可验证奖励的强化学习（RLVR）中，当正确性可以被自动检查时，后训练非常有效。但许多重要的模型行为需要同时满足多个定性标准。评分量奖励（Rubric-based Rewards）通过对每个标准打分并聚合为标量奖励来解决这个问题。

但作者发现：**标准静态聚合混淆了"人类认为重要的标准"和"当前能作为优化信号的标准"**。很多重要标准已经饱和或暂时不可达，而能区分 rollout 的标准不一定是人类权重最大的那些。

#### 核心方法

POW3R 引入了**策略感知**的奖励权重自适应：
- 保留人类权重和类别平衡作为评分目标
- 在训练过程中自适应调整标准级别的奖励权重
- 使用 rollout 级别的对比来强调当前能区分策略输出的标准
- 使 GRPO 奖励更具信息量，而不改变底层评估目标

#### 核心发现

| 指标 | POW3R | 标准 GRPO | 改进 |
|------|-------|----------|------|
| 均值评分奖励 | 更高 | 基准 | 30/30 比较中胜 24 |
| 严格完成率 | 更高 | 基准 | 显著提升 |
| 收敛速度 | 2.5-4x 更快 | 基准 | 大幅减少训练步数 |

在 3 个基础策略、2 个数据集（涵盖多模态和纯文本）上，POW3R 在 30 个基础策略/指标比较中赢得了 24 个。

#### 启发

如果你在做 RLHF/RLVR 训练，评分量奖励的设计不应该只是"人类觉得什么重要就给什么权重"。更重要的是区分：**"最终答案中什么应该重要"和"当前什么能教会策略"**。

#### 💡 对你的价值

对于正在进行模型后训练的团队，POW3R 提供了一种让 RL 训练更高效的方法。训练步数减少 2.5-4 倍意味着显著的算力节省。

### 2.3 OpenComputer：计算机使用 Agent 的可验证软件世界

**论文：** arXiv:2605.19769  
**作者：** Jinbiao Wei 等  
**日期：** 2026 年 5 月 19 日

#### 研究方法

OpenComputer 是一个面向计算机使用 Agent（Computer-Use Agents）的验证器接地框架。它整合了四个组件：

1. **应用特定状态验证器：** 为真实应用暴露结构化检查端点
2. **自我进化验证层：** 使用执行接地反馈提高验证器可靠性
3. **任务生成管道：** 合成真实且机器可检查的桌面任务
4. **评估框架：** 记录完整轨迹并计算可审计的部分信用奖励

#### 核心发现

当前形式下，OpenComputer 覆盖：
- **33 个桌面应用**（浏览器、办公工具、创意软件、开发环境、文件管理器、通讯应用）
- **1,000 个最终任务**

关键发现：
- 硬编码验证器比 LLM-as-Judge 评估更接近人类裁决
- 前沿 Agent 在端到端完成上仍然挣扎，尽管有局部进展
- 开源模型从 OSWorld-Verified 分数出现急剧下降，暴露了稳健计算机自动化的持续差距

#### 💡 对你的价值

如果你在做 Computer-Use Agent 的评估，OpenComputer 提供了比 LLM-as-Judge 更可靠的验证方案。它揭示了当前 Agent 的真正瓶颈——不是做不好单个操作，而是无法完成端到端的多步桌面任务。

### 2.4 分布自由的连续 AI Agent 评估不确定性量化

**论文：** arXiv:2605.19779  
**作者：** Megan Wang 等  
**日期：** 2026 年 5 月 19 日  
**发表：** ICML 2026 AgenticUQ 研讨会

#### 核心方法

将分裂共形预测（Split Conformal Prediction）和自适应共形推断（ACI）应用于连续 AI Agent 评估，为预测的质量评分提供分布自由覆盖保证。

#### 核心发现

- 共形区间在 24 小时预测范围内的校准误差低于 0.02
- ACI 在 Agent 发布后正确地将区间加宽 35%，然后重新收敛
- 开发了多 Agent 管道的组合不确定性边界
- 评估了 50 个 Agent，每 Agent 条件覆盖良好集中在标称水平附近（均值 80.4%）
- 跨来源情绪发散预测排名不稳定性（r=0.64, p<0.01）

#### 💡 对你的价值

如果你维护 Agent 排行榜或需要持续评估 Agent 性能，这篇论文提供了一种数学上严谨的不确定性量化方法，不依赖任何分布假设。

### 2.5 硬件感知代码优化中 LLM Agent 的先验 vs 搜索

**论文：** arXiv:2605.19782  
**作者：** Dmitry Redko 等  
**日期：** 2026 年 5 月 19 日

#### 研究方法

通过三个受控实验研究 LLM Agent 在硬件感知代码优化中的行为：
1. 纯黑盒优化
2. 零样本核生成
3. 反馈循环核优化

#### 核心发现

1. **在纯黑盒优化中，LLM 表现为贪婪优化器**——不会有效探索搜索空间
2. **在零样本核生成中，提供明确的输入大小信息没有可测量的效果**——模型无论大小或温度设置都收敛到相同的核参数
3. **在反馈循环核优化中，CUDA 在迭代反馈下单调改进，而 TVM IR 主动退化**——当模型操作低密度语言时核优化退化

**关键结论：** LLM 在代码优化任务中高度依赖于预训练先验，而非提供的反馈或 Agent 结构。

#### 💡 对你的价值

如果你在用 LLM 做代码优化（如 GPU 核函数调优），不要指望"给更多反馈就能让模型学得更好"。模型更依赖它已经学到的先验知识，而不是你给它的反馈。选择预训练中见过足够多相关内容的模型，比设计更复杂的 Agent 结构更有效。

### 5.3 ⭐ Probabilistic Tiny Recursive Model（arXiv:2605.19943）

#### 研究方法

Tiny Recursive Models (TRM) 通过迭代精炼潜在状态和最终答案来解决复杂推理任务，仅使用现代 LLM 参数的一小部分。但确定性递归可能导致收敛到次优解。

作者引入 **Probabilistic TRM (PTRM)**：
- 在每个深度递归步骤注入高斯噪声
- 允许并行轨迹探索不同的解空间
- 使用模型现有的 Q head 进行选择（用于早停）
- **无需重新训练或任务特定增强**

#### 核心发现

| 基准 | TRM | PTRM | 前沿 LLM | 成本比 |
|------|-----|------|---------|-------|
| Sudoku-Extreme | 87.4% | **98.75%** | — | — |
| Pencil Puzzle Bench | 62.6% | **91.2%** | 55.1% | <0.0001x |

PTRM 在 Pencil Puzzle Bench 上达到了前沿 LLM 近两倍的准确率，但成本不到其万分之一——仅使用 7M 参数。

#### 启发

这是一个"小模型+概率探索"击败"大模型+确定性推理"的典型案例。对于特定领域的推理任务（如数独、逻辑谜题），不需要万亿参数模型——精心设计的递归框架+随机探索就够了。

#### 💡 对你的价值

如果你的应用场景涉及结构化的推理问题（逻辑推理、数学证明、规划问题），PTRM 提供了一种极其高效的方法。7M 参数意味着可以在边缘设备上实时运行。
