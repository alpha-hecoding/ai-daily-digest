### 2.2 claude-context：让代码库成为 Claude Code 的上下文

**💡 对你的价值**：如果你使用 Claude Code 进行开发，这个工具可以显著提升代码理解和生成的准确性。安装即用：`npx @zilliztech/claude-context`。

### 2.4 Chat2Workflow：用自然语言生成可执行工作流

📄 **arXiv:2604.19667** — 《Chat2Workflow: A Benchmark for Generating Executable Visual Workflows with Natural Language》

**研究问题**：当前工业界大量使用可视化工作流（如 Dify、Coze），但构建这些工作流需要开发者手动设计节点、编写 prompt、反复调试。论文研究 LLM 能否自动化这个过程。

**研究方法**：
- 从真实商业工作流中构建大规模基准数据集
- 每个实例可转化为 Dify/Coze 等平台可直接部署的工作流
- 测试 SOTA LLM 自动生成工作流的能力

**核心发现**：
- SOTA 模型能理解高层意图，但**难以生成正确、稳定、可执行的工作流**
- 复杂需求下，生成失败率很高
- 论文提出的 Agent 框架最多提升了 5.34% 的解决率，但仍有巨大改进空间

**💡 对你的价值**：如果你在使用 Dify、Coze 等平台构建工作流，这篇论文确认了当前的痛点——自动生成工作流还不可靠。建议继续使用手动构建+LLM 辅助的方式。

### 2.5 ReTAS：用"正反合"辩证法解决多 Agent 认知偏差

📄 **arXiv:2604.19548**（ACL 2026 主会）— 《Taming Actor-Observer Asymmetry in Agents via Dialectical Alignment》

**核心发现**：多 Agent 系统中存在类似人类的**行动者-观察者不对称（AOA）**认知偏差：
- Agent 作为"行动者"反思自身时，倾向于将失败归因于外部因素
- Agent 作为"观察者"审计他人时，倾向于将相同错误归因为内部缺陷
- 简单切换视角会在超过 20% 的情况下触发 AOA 效应

**解决方案 ReTAS**：
- 基于"论题-反论题-综合"（Thesis-Antithesis-Synthesis）的辩证推理
- 使用 GRPO（Group Relative Policy Optimization）训练
- 引导 Agent 综合冲突视角，达成客观共识

**💡 对你的价值**：如果你在设计多 Agent 系统（如代码审查 Agent + 执行 Agent），ReTAS 揭示了当前框架的一个盲点。论文的方法论可以借鉴到你的 Agent 设计中。

## 五、值得深读的研究

### 5.1 EasyRL：仅需 10% 简单样本即可自我进化

📄 **arXiv:2604.18639**（ACL 2026 Findings）— 《Easy Samples Are All You Need: Self-Evolving LLMs via Data-Efficient Reinforcement Learning》

**研究背景**：当前 LLM 的强化学习（RL）后训练面临两大困境：
- 监督学习：标注成本极高
- 无监督学习：模型崩溃或奖励劫持

**研究方法**：
1. **热身阶段**：用少量（10%）简单标注数据进行监督 RL 初始化
2. **分而治之**：对未标注数据按不确定性分级——低不确定性用一致性选择，中等不确定性用反思解决
3. **难度递进自训练**：迭代伪标签 + RL 持续强化推理能力

**核心发现**：
- 仅需 **10% 的简单标注数据**即可超越 SOTA 基线
- 在数学和科学基准测试上持续领先
- 模拟了人类认知学习曲线：从简单到困难

**💡 对你的价值**：如果你在做 LLM 微调或 RL 训练，EasyRL 提供了一种数据高效的训练范式。大幅减少标注成本的同时保证性能。

### 5.2 TEMPO：推理时训练不再 plateau

📄 **arXiv:2604.19295** — 《Scaling Test-time Training for Large Reasoning Models》

**研究问题**：现有测试时训练（TTT）方法在推理时适应模型参数后很快达到性能 plateau，无法从额外推理计算中获益。

**原因诊断**：
- 自我生成的奖励信号随策略模型进化而漂移
- 导致性能停滞和多样性崩溃

**解决方案 TEMPO**：
- **交替策略**：在无标签问题上进行策略微调 + 在标注数据上定期校准 critic
- **理论框架**：通过 EM（期望最大化）算法形式化交替过程
- **发现**：先前方法可解释为省略了关键校准步骤的不完整变体

**实验结果**：
| 模型 | AIME 2024 基准 | TEMPO 提升 |
|------|---------------|-----------|
| OLMO3-7B | 33.0% → 51.1% | +18.1% |
| Qwen3-14B | 42.3% → 65.8% | +23.5% |

**💡 对你的价值**：如果你在使用 LLM 做数学推理或复杂推理任务，TEMPO 提供了一种无需额外训练即可在推理时持续提升性能的方法。

### 5.3 几何驱动 LoRA：用数学决定微调哪几层

📄 **arXiv:2604.19321** — 《Geometry-Driven Identification for Parameter-Efficient Adaptation in Large Language Models》

**研究问题**：LoRA 微调时，选择哪些层进行适配一直是个启发式决策。本文用几何方法给出训练-free 的层选择策略。

**创新方法**：
- 将隐藏状态的演化建模为高维几何轨迹
- 使用 **RDP 算法**（Ramer-Douglas-Peucker，多边形简化算法）识别关键断点
- 这些几何拐点直接决定哪些层应该被适配

**实验结果**（Qwen3-8B-Base，MMLU-Math）：
| 方法 | 适配层数 | 准确率 |
|------|----------|--------|
| 全 36 层适配 | 36 | 79.32% |
| 随机 13 层 | 13 | 75.56% |
| **RDP 选 13 层** | **13** | **81.67%** |
| 基线模型 | 0 | 74.25% |

**核心洞察**：仅用 13 个 RDP 选定的层（81.67%），性能超过全层适配（79.32%），同时减少 64% 的训练参数量。

**💡 对你的价值**：如果你在做 LoRA 微调，这篇论文提供了一种无需训练即可智能选择适配层的方法。实现简单（RDP 是经典算法），效果显著。

### 5.4 LLM 优化器的轨迹分析：什么让 LLM 成为好的优化器？
