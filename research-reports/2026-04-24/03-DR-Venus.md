# 🔬 深度研究报告：DR-Venus-4B —— 仅 10K 开放数据训练的边缘深度研究 Agent

> **论文**: Towards Frontier Edge-Scale Deep Research Agents with Only 10K Open Data (arXiv:2604.19859)  
> **作者**: Venus Team（浙江大学等）  
> **日期**: 2026-04-21  
> **模型/代码**: 已开源

---

## 一、一句话总结

浙江大学团队仅用约 **10,000 条开放数据**就训练出一个 4B 参数的深度研究 Agent（DR-Venus），在多个基准上**超越所有 9B 以下的 Agent 模型**，并大幅缩小与 30B 级系统的差距——**证明小模型+高质量数据+恰当训练策略 > 盲目堆参数**。

---

## 二、核心方法

### 2.1 两阶段训练

#### 阶段一：Agentic SFT（监督微调）

**数据清洗流水线**：
1. **环境对齐**：将 10,001 条原始轨迹转换为统一交互格式
2. **禁用工具裁剪**：移除不允许的工具调用（1,064 条轨迹受影响，移除 3,378 次调用）
3. **去重**：移除重复的搜索和浏览调用（6,821 条轨迹受影响，移除 15,728 次交互）
4. **正确性过滤**：用 Qwen3-235B 作为裁判，仅保留正确答案的轨迹 → 剩余 9,365 条
5. **Turn-aware 重采样**：对长轨迹加权重（0-50 turn: 1×, 51-100: 2×, 100+: 5×）→ 最终 18,745 条实例

**关键洞察**：>100 turn 的长轨迹比例从 13.29% 提升到 33.21%，显著提升模型在长程任务上的表现。

#### 阶段二：Agentic RL（强化学习）

**创新点**：
- 基于 **IGPO**（Information Gain Policy Optimization）
- 设计 **turn-level 奖励**（而非传统的轨迹级稀疏奖励）
- 奖励组成：**信息增益监督 + 格式感知正则化**
- 目的：增强监督密度和 turn-level 信用分配

**为什么对 4B 小模型有效？**
- 传统 RL 中，小模型的 rollout 组经常没有成功轨迹 → advantage collapse
- Turn-level 奖励提供每步的密集监督，避免训练崩溃
- 格式感知正则化保持行为稳定性

## 三、性能表现

### 3.1 核心基准

| 基准 | DR-Venus-4B | 对比 |
|------|-------------|------|
| BrowseComp | 显著优于 <9B 模型 | 缩小与 30B 级系统差距 |
| BrowseComp-ZH | 显著优于 <9B 模型 | 中文深度研究能力突出 |
| 多个深度研究基准 | 全面超越此前所有 9B 以下 Agent | 新 SOTA |

### 3.2 Pass@K 分析

- **4B Agent 的能力天花板已经很高**
- 单次采样可能不最优，但多次采样中能找到高质量解决方案
- **Test-time scaling**（测试时扩展）对小模型特别有效

### 3.3 工具使用分析

- 成功轨迹比失败轨迹**更多依赖浏览工具**
- RL 进一步校准了工具使用，朝向更有效的证据获取
- 搜索与浏览的平衡是长程任务成功的关键

## 四、深层意义

### 4.1 数据质量 > 数据数量

- 仅 10K 条数据，经过严格清洗和重采样后达到 18.7K 有效实例
- 小模型对噪声更敏感，数据质量比数量更重要
- 这与 LLM 预训练的"数据越多越好"范式不同

### 4.2 边缘部署的可行性

- **4B 模型可在消费级硬件运行**（单 GPU 甚至 CPU）
- 优势：低成本、低延迟、隐私保护
- 适用于：移动设备、IoT 设备、边缘服务器

### 4.3 训练策略的可迁移性

- Turn-level RL + IGPO 的方法可推广到其他 Agent 任务
- 长程任务（>50 turn）的训练需要特殊的监督设计
- 格式感知正则化是保持行为稳定的关键

---

## 五、对我们的启示

### 5.1 如果我们要训练小型 Agent

1. **数据优先**：不要追求数据量，追求数据质量
2. **长程重采样**：对需要多步推理的任务，确保训练集中有足够比例的长轨迹
3. **密集监督**：Turn-level 奖励比轨迹级奖励更适合小模型
4. **格式约束**：加入格式感知正则化，防止行为退化

### 5.2 如果我们要部署边缘 Agent

1. **4B 足够强**：不要默认需要 30B+ 的模型
2. **测试时扩展**：通过多次采样+选择来释放小模型潜力
3. **工具设计**：搜索+浏览是最核心的工具集，优先优化

### 5.3 与 SWE-chat 的关联

- SWE-chat 揭示 Agent 代码留存率仅 44%
- DR-Venus 展示了如何通过训练策略提升 Agent 可靠性
- **两者结合**：用 SWE-chat 的真实交互数据训练类似 DR-Venus 的 Agent，可能进一步提升编码 Agent 的代码留存率
